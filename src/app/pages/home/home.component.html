Welcome to MarvelService

MarvelService is an angular application based on what we learnt in Upgrade Hub's bootcamp.

The angular application contains guards, interceptors (spinner during http requests and switch http to https requests), reactive forms with built-in and custom validators. It also contains a login page, using localStorage to store the user's data, and a register form. Some functionality is still missing but it'll be added in future iteations.

MarvelService uses two APIs to request data from:

  • Marvel API: Main API to request information about Marvel Comic characters and related information
  • User API: custom node backend server to implement the user login/register functionality

Sections

MarvelService is composed of a common header and footer page plus the following sections:

  • Home: Landing page of the angular app
  • Gallery: Implementation of the gallery exercise with Marvel movie characters. Cards contain a title, subtitle, image and link. Cards are marked in golden rod colour when selected and a bottom-page counter counts the cards selected.
  • Characters: Main page of the application. It requests a list of comic characters to the Marvel API. It contains a pagination element to move between pages. Character cards are illuminated when hovering over them. Cards are also flipable. When available, it will display more information about the specific character. The back of the card contains a link to a detail view of the character with information about the comics, series, events and stories the character appears in.
  • Contact: The contact section contains a reactive form with built-in and custom validators. It does not offer more functionality than that

Usage

The main Characters section requires authentication. You can log in using the icon on the right-hand top corner. If you don't have an account yet you can register using the link in the 'Collapse' element (login icon).

When you register with MarvelService you are automatically signed in to the application. Your user data will be stored in the user backend service and in the browser's localStorage.
